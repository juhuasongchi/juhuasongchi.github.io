简单来说

src用于引入内容到前元素的位置（拿取）,href用于指出当前内容的来源（连结前往）。


src

  • source
    • n.根源，本源; 源头，水源; 原因; 提供消息的人;
    • vt.（从…）获得;发起;向…提供消息;寻求（尤指供货）的来源；
    • vi.原料来源;起源;寻求来源;寻求生产商（或提供商）;
  • 指向外部资源的位置，指向的内容将会嵌入到文档中当前标签所在位置；在请求src资源时会将其指向的资源下载并应用到文档内，例如js脚本，img图片和frame等元素：
    • <img src="image.jpg" />
      src 拿取 image.jpg 作为这个 img 的源档案，就是img标签显示 image.jpg ；

    • <script src ="js.js"></script>
      当浏览器解析到该元素时会暂停其他资源的下载和处理，直到将该资源加载、编译、执行完毕，图片和框架等元素也如此，类似于将所指向资源嵌入当前标签内。这也是为什么将js脚本放在底部而不是头部。


href

  • Hypertext Reference
    • n.参考;参考书;提及，涉及;证明人，介绍人;
    • v.引用;参照;
  • 指向网络资源所在位置，建立和当前元素（锚点）或当前文档（链接）之间的链接：
    • <a href="image.jpg" />
      href 跳转到 image.jpg，即是网址会跳转到 image.jpg；

    • <link href="common.css" rel="stylesheet"/>
      浏览器会识别该文档为css文件，会并行下载资源并且不会停止对当前文档的处理。这也是为什么建议使用link方式来加载css，而不是使用@import方式。
